  8. TLSA Records: The Async Problem

  This is the interesting technical challenge.

  The problem:
  1. Mitamae runs, deploys lego cert config
  2. Lego runs (via cron or hook), issues cert
  3. Need to update DNS with TLSA record from NEW cert
  4. But mitamae is done running

  Solutions:

  Option A: Lego renew hook
  # cookbooks/lego/default.rb

  # Create hook script
  file "/usr/local/bin/lego-tlsa-update.sh" do
    mode "0755"
    content <<~SH
      #!/bin/sh
      # Called by lego after cert renewal
      
      DOMAIN="$LEGO_CERT_DOMAIN"
      CERT="$LEGO_CERT_PATH"
      
      # Generate TLSA record
      TLSA=$(openssl x509 -in "$CERT" -pubkey -noout | \\
        openssl pkey -pubin -outform DER | \\
        openssl dgst -sha256 -binary | \\
        openssl enc -base64)
      
      # Update knot zone via knotc
      knotc zone-begin example.com
      knotc zone-unset example.com _443._tcp.${DOMAIN}. TLSA
      knotc zone-set example.com _443._tcp.${DOMAIN}. 3600 TLSA "3 1 1 ${TLSA}"
      knotc zone-commit example.com
    SH
  end

  # In lego cert resource:
  lego_cert "example.com" do
    renew_hook "/usr/local/bin/lego-tlsa-update.sh"
  end

  Option B: mitamae-triggered update
  # Calculate TLSA from existing cert at mitamae runtime
  # Update zone file template with current values
  # Requires cert to exist before mitamae runs

  cert_path = "/etc/ssl/example.com.crt"

  tlsa_hash = if ::File.exist?(cert_path)
    # Calculate TLSA record
    result = run_command("openssl x509 -in #{cert_path} -pubkey -noout | openssl pkey -pubin -outform DER | openssl dgst -sha256 -binary | openssl enc -base64")
    result.stdout.strip
  else
    nil
  end

  # Pass to knot zone template
  template "/var/knot/zones/example.com.zone" do
    variables(

  1. SPF for Outbound Mail

  You're correct. MX specifies incoming. For outbound, receiving servers check YOUR SPF record.

  Your current SPF (guessing):
  example.com. TXT "v=spf1 +mx -all"

  This means "servers listed in my MX records can send mail for this domain."

  This works IF:
  - Your outbound SMTP IP == your MX server IP
  - For you, it probably does (f0rk sends and receives)

  Google uses netblocks because:

      # Track offenses across runs
      # Format: IP OFFENSE_COUNT LAST_SEEN

      process_offender() {
        IP=$1
        REASON=$2

        # Increment offense count
        COUNT=$(grep "^$IP " "$STATEFILE" 2>/dev/null | awk '{print $2}')
        COUNT=${COUNT:-0}
        COUNT=$((COUNT + 1))

        # Update state
        grep -v "^$IP " "$STATEFILE" > "$STATEFILE.tmp" 2>/dev/null
        echo "$IP $COUNT $(date +%s) $REASON" >> "$STATEFILE.tmp"
        mv "$STATEFILE.tmp" "$STATEFILE"

        # Ban if 3+ offenses
        if [ "$COUNT" -ge 3 ]; then
          pfctl -t banned -T add "$IP" 2>/dev/null && \
            logger -t autoban "Banned $IP after $COUNT offenses ($REASON)"
        fi
      }

      # Parse SSH attempts
      tail -1000 /var/log/authlog | \
        grep "Failed password" | \
        awk '{print $(NF-3)}' | \
        while read IP; do
          process_offender "$IP" "SSH"

        grep -i "AUTH" | \

          process_offender "$IP" "HTTP-SCAN"
        done

      # Cleanup old state (7 days)
      NOW=$(date +%s)
      grep -v '^#' "$STATEFILE" | while read IP COUNT LASTSEEN REASON; do
        AGE=$((NOW - LASTSEEN))
        if [ "$AGE" -gt 604800 ]; then

● Yeah, You're Right

  Cron every 5 min for log parsing is janky polling. Event-driven is better.

  OpenBSD-Native Approach

  Option 1: syslog pipe (cleanest)

  OpenBSD's syslog can pipe to programs directly:

  # /etc/syslog.conf
  auth.info    |/usr/local/bin/watch-auth.sh
  mail.info    |/usr/local/bin/watch-mail.sh

  Then:

  #!/bin/sh
  # /usr/local/bin/watch-auth.sh

  while read LINE; do
    echo "$LINE" | awk '/Failed password/ {
      ip = $(NF-3)
      system("pfctl -t banned -T add " ip " 2>/dev/null")
      system("logger -t autoban \"Banned " ip " (SSH)\"")
    }'
  done

  Restart syslogd after adding pipes:
  rcctl restart syslogd

  This processes logs in real-time, zero polling.

  mkfifo /var/log/authpipe
  mkfifo /var/log/mailpipe
